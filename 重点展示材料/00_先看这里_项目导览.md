# 先看这里：项目导览

这是一个用于求职展示的 AI 统计报告生成 workflow prototype。**本项目面向需要频繁撰写数据报告的业务经理和社会科学研究者，目标是降低统计报告撰写中的重复劳动、格式错误和文本、表格、图表不一致风险。**

项目展示的是如何把高频、重复、容易出错的统计报告流程拆解成可复现的 Python workflow，并在关键统计决策处保留人工确认。**它是一个包含数据检查、统计决策、报告生成、图表输出和质量检查的端到端 workflow prototype。当前对外展示方式是简历中的 GitHub 链接。**

## 目标用户与使用场景

**目标用户：需要批量整理数据报告的业务经理、数据分析协作者和社会科学研究者。**

**核心场景：用户已有结构化数据和分析计划，希望 workflow 先完成统计适用性检查，并在人工审核确认后快速生成业务友好的统计报告、APA-style 均值表、结果图、适用性检查和质量检查。**

**核心痛点：人工整理统计结果耗时，且容易出现统计决策不准确、报告要素遗漏、文本、表格、图表数字不一致和格式不统一等问题。**

## MVP 验证结果

**本 prototype 使用 4 个合成测试情景进行初步验证，覆盖普通单因素方差分析、稳健方差分析、仅诊断输出和非显著结果克制表达。当前 MVP 在以下 6 项 workflow 稳定性指标上通过内部测试：**

- **任务完成率：100%**
- **决策路径准确率：100%**
- **人工确认合规率：100%**
- **报告完整率：100%**
- **文本、表格、图表一致率：100%**
- **统计格式通过率：100%**

**同时，基于 3 份可报告测试情景 Word 的人工审核记录和当前完成时间，补充 3 项产品化评估指标：**

- **人工审核修改率：8.52%**  
  按修订记录中“被修改原文字数 / 原文档总字数”计算，即 407 / 4,776。
- **统计结论可追溯率：100%**  
  关键统计结论均可回溯到 `analysis_results.json`、均值表、图表或 workflow 输出；业务建议和解释性判断不计入该指标。
- **时间节省与效率提升：节省 80 分钟，约 9.0x 速度提升**  
  4 个合成测试情景的 workflow 分析与报告约用 10 分钟，纯人工估计至少 90 分钟，时间减少 88.9%。

**由于测试样例数量仍较小，当前结果仅用于 MVP 可行性验证，不代表正式上线后的泛化表现。时间节省是当前 MVP 估算口径，不是正式基线实验。**

## 项目边界

- 所有数据均为合成数据或模拟数据。
- 第一版 MVP 只支持 between-subjects one-way ANOVA。
- **本项目不声称替代统计判断或论文作者责任，AI 生成内容必须经过人工审核后才能用于正式报告。**

## 技术与产品价值

### 产品价值

- 识别真实工作流中的重复任务：数据检查、统计分析、报告、表格、图、Word 输出和 QA。
- **将统计报告撰写从一次性人工整理，转化为可复用、可检查、可迭代的 workflow。**

### 技术实现

- 把单次 prompt 变成可复用 workflow。
- **用 Python workflow 固定关键处理步骤，降低纯自然语言生成带来的不稳定性。**
- **将最终报告、Markdown 预览、结果图、均值表和 QA 报告拆分为多个可检查产物。**

### AI 风险控制

- 用统计决策护栏避免 AI 盲目生成统计结论。
- 用测试情景展示 workflow 稳定性，而不是只展示一个成功样例。
- **在关键统计决策处保留人工确认，避免 AI 自动替代研究者判断。**
